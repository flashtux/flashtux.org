1. {% trans "My sound card is properly installed but not detected. What can I do?" %}

{% trans "Under Dos/Windows, run game choosing appropriate port and IRQ." %}
{% blocktrans %}Example: "ftris -port220 -irq5" if sound card is on port 220h and IRQ 5.{% endblocktrans %}
{% trans "Under Dos/Windows, only SoundBlaster or 100% compatible are supported." %}
{% trans "Under Windows 9x, install SB16 Emulation driver (in case of necessity)." %}
{% trans "Under Windows 2000, install SoundFX 2000 or VdmSound." %}
{% trans "Under Linux, all cards are supported." %}

{% blocktrans %}Otherwise, run game without sound/music: "ftris -nosound"{% endblocktrans %}

3. {% trans "Game hangs up during intro." %}

{% blocktrans %}Disable introduction: "ftris -nointro" (and see FAQ #7).{% endblocktrans %}

4. {% blocktrans %}Game hangs up with error message "svgalib: Cannot get I/O permissions" (Linux only){% endblocktrans %}

{% blocktrans %}Svgalib requires "root" privileges to access video card.{% endblocktrans %}
{% blocktrans %}You must be connected as "root" user to run FlashTris or you must activate "setuid" bit on executable (issue "chmod +s ./ftris" command as "root" user).{% endblocktrans %}

5. {% blocktrans %}Game hangs up with error message "svgalib: mmap error in paged screen memory" (Linux only){% endblocktrans %}

{% trans "This is an error of svgalib with kernels 2.3.x and 2.4.x." %}
{% blocktrans %}Download last version of svgalib with sources (here: https://www.svgalib.org) and compile it without option "BACKGROUND = Y" in file Makefile.cfg (comment this line).{% endblocktrans %}

6. {% trans "Menu and/or game are very slow." %}

{% blocktrans %}In "advanced" menu (under burning title): disable menu's flames and/or motion pixels.{% endblocktrans %}
{% trans "With 80486 processor or less, it is advisable to install a pedal (and to pedal fast!)." %}
